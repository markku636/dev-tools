## 1. 目標與核心價值

打造跨平台桌面應用，以單一介面整合管理多種主流資料庫，取代多個零散工具。

- **一站式管理**：多種資料庫共用同一套介面、連線管理與主題。
- **美觀現代**：深色為主、可切換亮色，依資料庫類型色標區分。
- **輕量高效**：採用 Tauri 架構，記憶體佔用遠低於 Electron 同類產品。
- **安全可靠**：連線憑證以作業系統 keychain 加密儲存（規劃中），備份檔可選 AES 加密。

## 2. 設計範式分類

多種資料庫分屬三種資料範式，UI 與操作邏輯依範式分流：

| 範式 | 資料庫 | 資料模型 | 查詢方式 |
|------|--------|----------|----------|
| 關聯式 | MySQL, PostgreSQL, SQLite | 表格 / 列 / 欄 | SQL |
| 文件型 | MongoDB | 集合 / 文件 (BSON) | Query / Aggregation |
| 鍵值型 | Redis | Key + 多種資料結構 | Redis 命令 |

## 3. 技術選型

| 項目 | 選擇 | 理由 |
|------|------|------|
| 應用框架 | Tauri 2 (Rust + Web) | 比 Electron 輕量約 10 倍 |
| 前端 | React 18 + TypeScript + Vite | 生態成熟、型別安全 |
| UI | Tailwind CSS（shadcn/ui 風格） | 美觀、可客製化 |
| 狀態 | Zustand | 輕量，適合多連線狀態 |
| 後端驅動 | sqlx (MySQL/PG/SQLite)、後續 mongodb、redis | 統一介面、連線池 |

將資料庫連線統一放在 Rust 後端，前端透過 Tauri command 呼叫，避免前端直連，兼顧安全與效能。

## 4. 功能藍圖

依範式提供不同的瀏覽與編輯體驗：

| 範式 | 樹狀結構 | 資料檢視 |
|------|----------|----------|
| 關聯式 | 連線 → 資料庫/schema → 表 → 欄位 | 表格化瀏覽，分頁、結構/資料分頁、儲存格編輯 |
| 文件型 | 連線 → 資料庫 → 集合 | 文件攤平成表格（已實作）；JSON 巢狀值內嵌呈現 |
| 鍵值型 | 連線 → DB(0-15) → key | key 列表化為表格，雙擊開五種結構詳情（已實作） |

# Navicat 操作習慣復刻

互動設計以全面復刻 Navicat 操作習慣為目標，讓使用者的肌肉記憶可無痛轉移。各資料庫統一遵循「開啟 → 表格檢視 → 底部導覽 → 儲存格直接編輯 → ✓ 套用變更」的一致流程。

## 主視窗與工具列

- 左側「連線/物件樹」常駐，最頂層為連線；**雙擊連線**才展開並建立連線。
- 上方一排**大圖示工具列**（連線、表、查詢、備份…），復刻 Navicat 標誌性的視覺風格。
- **雙擊**資料庫/表 → 直接於右側主區開啟分頁（非單擊）。

## 資料表格檢視（核心習慣）

- **雙擊表 → 直接進入資料表格檢視**。
- **底部導覽列**：第一頁 / 上一頁 / 下一頁 / 最後一頁。
- **儲存格直接編輯**：雙擊儲存格進入編輯，修改中的儲存格變色（琥珀底），按 **✓ 套用** 才真正寫入資料庫，**✗ 捨棄** 取消所有待套用變更。
- Enter 套用單格、Esc 取消、可一鍵設為 NULL。

> 已實作（關聯式）。寫入以主鍵定位列；無主鍵的表為唯讀並於底部提示。

## 結構 / 資料 分頁切換

選取一個物件後，主區上方提供分頁，在同一物件的不同檢視間切換：

- **資料**：分頁式表格瀏覽 + 儲存格編輯。
- **結構**：欄位定義（欄名、型別、可空、鍵、預設、額外）。

## 查詢編輯器（上下分割 + F6）

- 「查詢」分頁開一個編輯區，**上半 SQL、下半結果**。
- **F6 或 ▶ 執行**查詢。

## 非關聯式的表格手感對應（已實作）

| 資料庫 | 對應方式 |
|--------|----------|
| MongoDB | 掃描一批文件，聯集所有欄位攤平成表格；雙擊集合直接進表格；巢狀欄位切 JSON 編輯 |
| Redis | 所有 key 列表化為主表格（key/型別/TTL/記憶體）；雙擊 key 進該結構細部編輯 |

關聯式（MySQL/PG/SQLite）原汁原味復刻；文件型與鍵值型借用相同外殼與導覽手感，資料區換成適合各自範式的元件。

## 色標系統

側欄連線依類型上色，一眼辨識：

- MySQL 藍 `#3b82f6`
- PostgreSQL 靛 `#6366f1`
- MongoDB 綠 `#22c55e`
- Redis 紅 `#ef4444`
- SQLite 橘 `#f59e0b`
